## ✨ Features

| Feature | Details |
|---|---|
| **Document Ingestion** | Load `.txt` and `.pdf` files from `/data`; chunk, embed, and persist to FAISS |
| **Semantic Retrieval** | Top-K similarity search surfaces the most relevant context per query |
| **Grounded Answers** | System prompt constrains the LLM to answer *only* from retrieved context |
| **Source Citations** | Every response shows which documents (file + page) were used |
| **Conversation Memory** | Sliding-window memory keeps multi-turn context without blowing the token budget |
| **Chat UI** | Clean dark-mode Streamlit interface with animated message bubbles |
| **Re-ingestion** | One-click knowledge base refresh from the sidebar |
| **Error Handling** | Graceful handling of missing API keys, empty `/data`, and runtime failures |
| **Modular Design** | `ingest.py`, `rag_pipeline.py`, `utils/`, and `app.py` are fully decoupled |

## 🚀 Setup & Installation

### Prerequisites

- Python 3.11+
- An [OpenAI API key](https://platform.openai.com/account/api-keys)

### 1 — Clone and install dependencies

```bash
git clone https://github.com/yourname/rag-chatbot.git
cd rag-chatbot
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2 — Configure environment

```bash
cp .env.example .env
# Open .env and set your API key:
# OPENAI_API_KEY=sk-...
```

### 3 — Add your knowledge base

Drop `.txt` or `.pdf` files into the `/data` folder. A sample FAQ is included to get you started immediately.

### 4 — Build the vector index

```bash
python ingest.py
```

Expected output:
```
10:42:01 [INFO] ═══ Starting document ingestion ═══
10:42:01 [INFO] Loaded 1 .txt document(s).
10:42:01 [INFO] Split into 28 chunk(s) (size=1000, overlap=200).
10:42:03 [INFO] Vector store saved to 'vectorstore'.
10:42:03 [INFO] ═══ Ingestion complete (28 chunk(s) indexed) ═══
```

### 5 — Launch the app

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## 💬 Example Queries

Once running, try questions like:

- *"How do I reset my password?"*
- *"What payment methods do you accept?"*
- *"Does the app support two-factor authentication?"*
- *"What's your refund policy?"*
- *"Can I export my data?"*

## 🔄 Updating the Knowledge Base

1. Add / remove / edit files in `/data`.
2. Click **"Re-ingest Documents"** in the sidebar, **or** run:
   ```bash
   python ingest.py
   ```
3. Reload the browser tab.

## ⚙️ Configuration Reference

| Variable | File | Default | Description |
|---|---|---|---|
| `OPENAI_API_KEY` | `.env` | — | Your OpenAI secret key |
| `CHUNK_SIZE` | `ingest.py` | `1000` | Characters per chunk |
| `CHUNK_OVERLAP` | `ingest.py` | `200` | Overlap between chunks |
| `LLM_MODEL` | `rag_pipeline.py` | `gpt-4o-mini` | OpenAI chat model |
| `EMBEDDING_MODEL` | `rag_pipeline.py` | `text-embedding-3-small` | Embedding model |
| `TOP_K_RETRIEVAL` | `rag_pipeline.py` | `4` | Chunks retrieved per query |
| `MEMORY_WINDOW` | `rag_pipeline.py` | `10` | Conversation turns in memory |
| `TEMPERATURE` | `rag_pipeline.py` | `0.2` | LLM temperature (0=deterministic) |

## 🛡️ Design Decisions

**Why FAISS over a hosted vector DB?**  
FAISS runs entirely locally — no extra service, no cost, no latency overhead. For a team that needs cloud-scale retrieval, swap `FAISS` for `Pinecone` or `Weaviate` in `ingest.py` and `rag_pipeline.py` without changing any other file.

**Why GPT-4o-mini?**  
It's fast, cheap, and more than capable for factual Q&A over retrieved context. The strict system prompt does the heavy lifting — the model's job is structured extraction, not creative generation.

**Why ConversationalRetrievalChain?**  
It handles the "condense question → retrieve → answer" loop automatically and integrates with LangChain memory cleanly. For production, consider replacing it with LCEL (LangChain Expression Language) for more granular control.
